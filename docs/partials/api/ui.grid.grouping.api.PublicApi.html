PublicApi (api in module ui.grid.grouping )

Description

Public Api for grouping feature

Methods

  • aggregateColumn(columnName, aggregationType)

    Sets the aggregation type on a column, if the column is currently grouped then it removes the grouping first. If the aggregationType is null then will result in the aggregation being removed

    Parameters

    • columnName – {string} –

      the column we want to aggregate

    • aggregationType – {string} –

      one of the recognised types from uiGridGroupingConstants

  • clearGrouping()

    Clear any grouped columns and any aggregations. Doesn't remove sorting, as we don't know whether that sorting was added by grouping or was there beforehand

  • collapseAllRows()

    collapse all grouped rows

  • collapseRow(row)

    collapse all children of the specified row. When you expand the row again, all grandchildren will retain their state

    Parameters

    • row – {gridRow} –

      the row you wish to collapse

  • collapseRowChildren(row)

    collapse all children of the specified row. When you expand the row again, all grandchildren will be collapsed

    Parameters

    • row – {gridRow} –

      the row you wish to collapse

  • expandAllRows()

    Expands all grouped rows

  • expandRow(row)

    expand the immediate children of the specified row

    Parameters

    • row – {gridRow} –

      the row you wish to expand

  • expandRowChildren(row)

    expand all children of the specified row

    Parameters

    • row – {gridRow} –

      the row you wish to expand

  • getGrouping(getExpanded)

    Get the grouping configuration for this grid, used by the saveState feature Returned grouping is an object { grouping: groupArray, aggregations: aggregateArray, expandedState: hash } where grouping contains an array of objects: { field: column.field, colName: column.name, groupPriority: column.grouping.groupPriority } and aggregations contains an array of objects: { field: column.field, colName: column.name, aggregation: column.grouping.aggregation } and expandedState is a hash of the currently expanded nodes

    The groupArray will be sorted by groupPriority.

    Parameters

    • getExpanded – {boolean} –

      whether or not to return the expanded state

    Returns

    {object} –

    grouping configuration

  • groupColumn(columnName)

    Adds this column to the existing grouping, at the end of the priority order. If the column doesn't have a sort, adds one, by default ASC

    This column will move to the left of any non-group columns, the move is handled in a columnProcessor, so gets called as part of refresh

    Parameters

    • columnName – {string} –

      the name of the column we want to group

  • setGrouping(config)

    Set the grouping configuration for this grid, used by the saveState feature, but can also be used by any user to specify a combined grouping and aggregation configuration

    Parameters

    • config – {object} –

      the config you want to apply, in the format provided out by getGrouping

  • toggleRowGroupingState(row)

    call expand if the row is collapsed, collapse if it is expanded

    Parameters

    • row – {gridRow} –

      the row you wish to toggle

  • ungroupColumn(columnName)

    Removes the groupPriority from this column. If the column was previously aggregated the aggregation will come back. The sort will remain.

    This column will move to the right of any other group columns, the move is handled in a columnProcessor, so gets called as part of refresh

    Parameters

    • columnName – {string} –

      the name of the column we want to ungroup
