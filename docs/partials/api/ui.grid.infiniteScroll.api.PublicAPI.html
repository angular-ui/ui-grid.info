object in module ui.grid.infiniteScroll

Description

Public API for infinite scroll feature

Methods

  • dataLoaded(scrollUp, scrollDown)

    Call this function when you have loaded the additional data requested. You should set scrollUp and scrollDown to indicate whether there are still more pages in each direction.

    If you call dataLoaded without first calling saveScrollPercentage then we will scroll the user to the start of the newly loaded data, which usually gives a smooth scroll experience, but can give a jumpy experience with large infiniteScrollRowsFromEnd values, and on variable speed internet connections. Using saveScrollPercentage as demonstrated in the tutorial should give a smoother scrolling experience for users.

    See infinite_scroll tutorial for example of usage

    Parameters
    ParamTypeDetails
    scrollUpboolean

    if set to false flags that there are no more pages upwards, so don't fire any more infinite scroll events upward

    scrollDownboolean

    if set to false flags that there are no more pages downwards, so don't fire any more infinite scroll events downward

    Returns
    promise

    a promise that is resolved when the grid scrolling is fully adjusted. If you're planning to remove pages, you should wait on this promise first, or you'll break the scroll positioning

  • dataRemovedBottom(grid, scrollUp, scrollDown)

    Adjusts the scroll position after you've removed data at the bottom. You should have called saveScrollPercentage before you remove the data, and if you're doing this in response to a needMoreData you should wait until the promise from loadData has resolved before you start removing data

  • resetScroll(scrollUp, scrollDown)

    Call this function when you have taken some action that makes the current scroll position invalid. For example, if you're using external sorting and you've resorted then you might reset the scroll, or if you've otherwise substantially changed the data, perhaps you've reused an existing grid for a new data set

    You must tell us whether there is data upwards or downwards after the reset

    Parameters
    ParamTypeDetails
    scrollUpboolean

    flag that there are pages upwards, fire infinite scroll events upward

    scrollDownboolean

    flag that there are pages downwards, so fire infinite scroll events downward

    Returns
    promise

    promise that is resolved when the scroll reset is complete

  • saveScrollPercentage()

    Saves the scroll percentage and number of visible rows before you adjust the data, used if you're subsequently going to call dataRemovedTop or dataRemovedBottom

Events

  • needLoadMoreData

    This event fires when scroll reaches bottom percentage of grid and needs to load data

  • needLoadMoreDataTop

    This event fires when scroll reaches top percentage of grid and needs to load data
